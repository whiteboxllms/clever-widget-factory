## Key Components

### Frontend (React)
- **SariSariChat.tsx**: Main chat interface
- **apiService**: HTTP client with auth
- **TanStack Query**: Data fetching and caching

### Backend (Lambda)
- **SearchPipeline**: Main orchestrator
- **QueryRewriter**: Intent extraction
- **FilterMapper**: SQL filter generation
- **HybridRetriever**: Semantic + filtered search
- **ResultFormatter**: Data transformation
- **ResponseGenerator**: Conversational AI

### AWS Services
- **API Gateway**: REST API endpoint
- **Lambda**: Serverless compute
- **Bedrock**: Claude 3.5 + Titan embeddings
- **RDS PostgreSQL**: Database with pgvector
- **Cognito**: Authentication

## Request Flow

1. User types message in chat
2. Frontend sends POST to `/api/sari-sari/chat`
3. API Gateway authenticates via Cognito
4. Lambda receives request with org context
5. SearchPipeline executes 5 steps
6. Response returned to frontend
7. Products displayed in chat interface

## Performance

- **Average Response Time**: 2-3 seconds
- **Embedding Generation**: ~500ms
- **Semantic Search**: ~200ms
- **Response Generation**: ~1s
- **Total Pipeline**: ~2s

## Scalability

- Lambda auto-scales with demand
- RDS connection pooling
- Bedrock handles concurrent requests
- Frontend caching via TanStack Query
